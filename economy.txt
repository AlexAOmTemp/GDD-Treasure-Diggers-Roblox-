1. Economy Overview

Экономика игры строится вокруг следующего цикла:

Break chests → Earn coins → Buy eggs → Hatch pets → Upgrade pets → Unlock islands → Optional Rebirth → Loop

Основная валюта игры — Coins.

Монеты используются для:

покупки яиц

открытия новых островов

доступа в Rare Areas

Все числовые параметры хранятся в economy.xlsx.

Все расчёты выполняются на сервере.

2. Income System

Основной источник дохода игрока — разрушение сундуков.

Доход зависит от:

базовой награды сундука

редкости сундука

экипированных питомцев

уровня питомцев

множителя острова

Формула дохода:

FinalIncome = BaseIncome * IslandMultiplier * (1 + RebirthBonus) * (1 + SumPetMultipliers) * (1 + min(FriendsBonus + AFKBonus, MaxBonus))

где SumPetMultipliers — сумма множителей экипированных питомцев (с учётом уровня и Golden), MaxBonus = 100% (конфиг).

Базовые значения находятся в:

economy.xlsx
sheet: Chests_Islands
Income Modifiers
AFK Income

Игрок может получать доход только в специальной AFK-зоне.

Особенности:

процент от обычного дохода

действует только в AFK зоне

имеет ограничение по времени

Параметры:

economy.xlsx
sheet: GlobalParams
Friends Bonus

Игрок получает дополнительный доход за друзей на сервере.

Особенности:

бонус за каждого друга

имеет максимальный лимит 100%

Параметры:

economy.xlsx
sheet: GlobalParams
Soft Cap

Для предотвращения гиперинфляции применяется Soft Cap к сумме множителей питомцев и бонусов Rebirth.

Soft Cap уменьшает эффективность слишком больших сумм множителей.

Формула хранится в:

economy.xlsx
sheet: IncomeBalance

Soft Cap для Rebirth: после определённого количества Rebirth прирост уменьшается (см. RebirthCurve).

Порог слияния одинаков для всех редкостей.

можно использовать питомцев разных уровней

5. Pity System

Pity System защищает игрока от длинной серии неудачных открытий яиц.

Особенности:

каждый тип яйца имеет свой pity counter

шанс Legendary увеличивается после неудачных попыток

после получения Legendary счётчик сбрасывается

Шанс увеличивается линейно до максимума (конфиг). Сброс счетчика происходит только при получении легендарки из этого же типа яйца. Получение легендарки из другого типа яйца не влияет на счетчик данного яйца.

Параметры находятся в:

economy.xlsx
sheet: Eggs
Data Persistence

Pity counters хранятся в профиле игрока.

Пример:

pityCounters = {
 starterEgg = 12,
 forestEgg = 4
}

Обновление происходит атомарно во время HatchEvent. В Bulk Open pity увеличивается за каждое неудачное открытие, но при откате транзакции pity counters также восстанавливаются до исходных значений.

Данные находятся в:

economy.xlsx
sheet: Eggs
Bulk Open

Игрок может открывать яйца пакетами:

1

3

8

Bulk Open не даёт ценовой скидки.

Стоимость равна открытию яиц по одному.

Атомарность: операция открытия нескольких яиц должна быть атомарной — либо открываются все яйца, либо ни одного. При ошибке сохранения все изменения откатываются, игроку возвращаются монеты, клиенту отправляется TransactionFailed. Откат выполняется атомарно: все изменения (монеты, питомцы, pity counters) отменяются одновременно, как если бы операция не выполнялась.

Egg Cooldown

Для предотвращения перегрузки сервера используется минимальный интервал между открытиями яиц.

Значение задаётся в:

economy.xlsx
sheet: GlobalParams

Дополнительно: RateLimit на запросы открытия яиц.

7. Rare Area Access

Rare Area — зона с повышенными наградами.

Игрок может купить временный доступ.

После покупки игрок получает:

доступ в Rare Area

бонус к награде сундуков

Стоимость и длительность:

economy.xlsx
sheet: GlobalParams
Access Timer

Используется Dirty Timer.

Таймер продолжает идти даже если игрок покинул игру.

В профиле игрока хранится:

rareAreaAccessUntil = timestamp

При входе в зону проверять доступ; если истёк — телепортировать.

Для предотвращения спама входов/выходов установлен серверный кулдаун (1 сек).

8. Auto Delete / Clean Up

Игрок может автоматически удалять ненужных питомцев.

Auto Delete можно настроить по редкости.

Удаление питомцев не даёт награды.

При добавлении нового питомца (хэтч, фьюжн) проверять настройки: если включено автоудаление для данной редкости и инвентарь полон или после достижения лимита — удалить питомца по приоритету: сначала Common, затем Rare, затем наименьший уровень, затем самого старого. Приоритет удаления используется как для автоудаления, так и для выбора питомцев на удаление при переполнении инвентаря.

11. Dev Products / Монетизация

Dev Products позволяют покупать монеты за Robux.

Используется динамическое ценообразование, которое масштабируется относительно прогресса игрока.

Дополнительно предлагаются эксклюзивные предметы:
- Уникальные питомцы (только за Robux)
- Бусты (например, Double Coins на 24 часа)
- Доступ в VIP-зону с повышенными наградами

Параметры:

economy.xlsx
sheet: DevProducts

Влияние на прогрессию:
- Покупка монет не даёт мгновенно перейти на следующий остров, если не выполнены другие условия (например, наличие питомцев определённого уровня).
- Цены растут с прогрессом игрока (динамическое ценообразование), что сглаживает эффект «pay-to-win».

12. Урон игрока и HP сундуков

Базовый урон игрока за клик равен 1. Множители питомцев и бонусы применяются к доходу, но не к урону по сундукам (кроме Mega Chest, где урон ограничен Damage Cap). HP сундуков на втором и последующих островах увеличивается пропорционально множителю острова и среднему доходу игрока. Точные значения в Chests_Islands.

14. Balance Simulation

Экономика балансируется через симуляции.

Минимальная проверка:

20 циклов Rebirth

Симуляции включают:
- Одиночная игра с разными комбинациями питомцев.
- Мультиплеер (несколько игроков, совместный урон по Mega Chest).
- Использование AFK и Friends бонусов.
- Покупки Dev Products.
- Разные стратегии (активный кликер vs AFK).

Симуляции должны включать стресс-тесты с максимальными значениями бонусов (макс. друзья, макс. уровень питомцев, все бонусы одновременно) и сценарии с несколькими игроками, чтобы убедиться в отсутствии экономических аномалий.

Симуляции выполняются в:

economy.xlsx
sheet: Simulation
